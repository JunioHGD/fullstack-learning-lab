# BACKEND-09 — Segurança de Backend e APIs

## Objetivo

Introduzir segurança de APIs como competência transversal: HTTPS, OWASP, validação, cookies, headers, CORS, segredos e logging seguro.

## Pré-requisitos

BACKEND-08.

## Progressão didática

Começa pela proteção de tráfego e riscos reconhecidos, depois aplica controles concretos em API, HTTP e operação.

| Código | Tópico | Pasta | Aulas |
|---|---|---|---:|
| 09.01 | HTTPS e segurança por padrão | `09-01-https-secure-defaults` | 2 |
| 09.02 | OWASP para aplicações web e APIs | `09-02-owasp-web-api-risks` | 2 |
| 09.03 | Validação, erros e exposição de dados | `09-03-validation-error-data-exposure` | 2 |
| 09.04 | Cookies seguros, security headers e CORS | `09-04-secure-cookies-headers-cors` | 2 |
