# Lessons — 00.02 — Arquitetura cliente-servidor e ciclo request-response

## Objetivo do tópico

Compreender como clientes e servidores se comunicam e como o ciclo de requisição e resposta organiza a web.

---

## 00.02.01 — Arquitetura cliente-servidor

### Objetivo

Diferenciar cliente, servidor e responsabilidades de cada lado da aplicação.

### Pré-requisitos

BACKEND-00.01.

### Resultado esperado

Identificar o papel do cliente e do servidor em um fluxo web.

### Conceitos para estudar

- Entenda o conceito central de **Arquitetura cliente-servidor**.
- Relacione o conceito com o fluxo de uma aplicação backend real.
- Identifique onde esse conhecimento aparece em APIs, serviços, banco de dados, segurança, testes ou deploy.
- Registre exemplos curtos e dúvidas no próprio processo de estudo.

### Observações

Use esta seção como base de estudo. A explicação detalhada, exemplos de código e anotações pessoais podem ser adicionados gradualmente conforme o tópico for estudado.
