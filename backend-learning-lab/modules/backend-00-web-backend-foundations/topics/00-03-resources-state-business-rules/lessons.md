# Lessons — 00.03 — Recursos, estado e regras de negócio

## Objetivo do tópico

Introduzir recursos, estado e regras de negócio como responsabilidades centrais do backend.

---

## 00.03.01 — Recursos e regras de negócio

### Objetivo

Entender como recursos representam entidades manipuladas pela aplicação e como regras controlam operações.

### Pré-requisitos

BACKEND-00.02.

### Resultado esperado

Diferenciar recurso, dado e regra de negócio.

### Conceitos para estudar

- Entenda o conceito central de **Recursos e regras de negócio**.
- Relacione o conceito com o fluxo de uma aplicação backend real.
- Identifique onde esse conhecimento aparece em APIs, serviços, banco de dados, segurança, testes ou deploy.
- Registre exemplos curtos e dúvidas no próprio processo de estudo.

### Observações

Use esta seção como base de estudo. A explicação detalhada, exemplos de código e anotações pessoais podem ser adicionados gradualmente conforme o tópico for estudado.
