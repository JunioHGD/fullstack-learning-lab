# Lessons — 05.01 — Validação e normalização de entrada

## Objetivo do tópico

Garantir que dados recebidos pela API sejam válidos e previsíveis.

---

## 05.01.01 — Validação de entrada no servidor

### Objetivo

Entender por que toda entrada deve ser validada no backend.

### Pré-requisitos

BACKEND-04.

### Resultado esperado

Explicar por que validação frontend não substitui validação backend.

### Conceitos para estudar

- Entenda o conceito central de **Validação de entrada no servidor**.
- Relacione o conceito com o fluxo de uma aplicação backend real.
- Identifique onde esse conhecimento aparece em APIs, serviços, banco de dados, segurança, testes ou deploy.
- Registre exemplos curtos e dúvidas no próprio processo de estudo.

### Observações

Use esta seção como base de estudo. A explicação detalhada, exemplos de código e anotações pessoais podem ser adicionados gradualmente conforme o tópico for estudado.
