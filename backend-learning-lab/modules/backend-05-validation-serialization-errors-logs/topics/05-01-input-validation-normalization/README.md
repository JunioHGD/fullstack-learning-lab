# 05.01 — Validação e normalização de entrada

## Módulo

BACKEND-05 — Validação, Serialização, Erros e Logs

## Objetivo do tópico

Garantir que dados recebidos pela API sejam válidos e previsíveis.

## Aulas do tópico

| Código | Aula | Objetivo | Resultado esperado |
|---|---|---|---|
| 05.01.01 | Validação de entrada no servidor | Entender por que toda entrada deve ser validada no backend. | Explicar por que validação frontend não substitui validação backend. |
| 05.01.02 | Normalização de dados recebidos | Preparar dados recebidos para uso consistente na aplicação. | Reconhecer casos simples de normalização de dados. |

## Arquivos deste tópico

- `lessons.md`: explicação organizada dos conceitos do tópico.
- `assimilation-questions.md`: perguntas para fixação e revisão ativa.
- `exercises/`: espaço para exercícios, códigos, experimentos e práticas.
