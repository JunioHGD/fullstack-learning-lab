# HTML-06 — Acessibilidade e Boas Práticas

## Objetivo do módulo

Consolidar HTML profissional com acessibilidade, navegação por teclado, foco, semântica, validação e segurança básica.

## Pré-requisitos

HTML-00 a HTML-05.

## Tópicos do módulo

- **06.01 — Texto alternativo e imagens acessíveis**: Aprofundar o uso de alt em imagens informativas e decorativas.
- **06.02 — Ordem de leitura e foco**: Garantir que a ordem do HTML faça sentido para leitura e navegação.
- **06.03 — Navegação por teclado**: Testar a navegação básica por teclado e identificar problemas.
- **06.04 — ARIA introdutório**: Entender ARIA como complemento, não substituto, de HTML semântico.
- **06.05 — Landmarks semânticas**: Reforçar regiões de página como mecanismos estruturais de acessibilidade.
- **06.06 — Formulários acessíveis**: Revisar formulários sob critérios de acessibilidade.
- **06.07 — Elementos interativos nativos**: Priorizar elementos HTML nativos para interações acessíveis.
- **06.08 — Boas práticas de marcação**: Escrever HTML limpo, semântico e manutenível.
- **06.09 — Segurança básica em links e iframes**: Aplicar cuidados básicos em links externos e conteúdo incorporado.
- **06.10 — Validação e conformidade**: Consolidar uma checklist de revisão profissional de HTML.

## Como usar este módulo

- Use `checklist.md` para acompanhar o progresso.
- Use `interview-questions.md` para revisar o módulo de forma integrada.
- Acesse cada tópico em `topics/`.
- Em cada tópico, estude `lessons.md`, responda `assimilation-questions.md` e pratique em `exercises/`.
